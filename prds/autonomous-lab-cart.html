Project Requirement Document

Dream Game

A team-developed Unity game prototype focused on responsive gameplay systems, procedural arm movement, and collision-based combat interactions.

Problem

The project needed gameplay systems that felt responsive and expressive while still being technically manageable within a team-based prototype scope.

Objective

Build a playable prototype with procedural motion, reliable combat interactions, and a technical architecture that supports team collaboration inside Unity.

Functional Scope

  • Implement vector-based inverse kinematics for arm movement
  • Support animation logic tied to gameplay behavior
  • Handle collision-based combat with hit detection and attack interactions
  • Maintain clear technical ownership within a team project workflow

Tools

  • Unity
  • C#
  • Animation systems
  • Collision and gameplay logic

Success Criteria

  • Playable combat behavior with consistent hit detection
  • Procedural limb movement that supports game feel
  • A stable prototype delivered through team collaboration
